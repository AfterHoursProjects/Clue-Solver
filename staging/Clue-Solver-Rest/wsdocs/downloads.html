C Client Library

Created Aug 18, 2012 3:48:24 PM

Introduction

The C module generates the source code for the ANSI-C-compatible data structures and (de)serialization functions that can be used in conjunction with libxml2 to (de)serialize the REST resources as they are represented as XML data.

The generated C source code depends on the XML Reader API and the XML Writer API as well as the <time.h>, <string.h>, and <stdlib.h> C standard libraries.

REST XML Example

#include <Clue-Solver-Rest.c> //... xmlTextReaderPtr reader = ...; //set up the reader to the url. Clue_Solver_Rest_ns0_clueServerStatus *response_element = ...; response_element = xml_read_Clue_Solver_Rest_ns0_clueServerStatus(reader); //handle the response as needed... //free the Clue_Solver_Rest_ns0_clueServerStatus free_Clue_Solver_Rest_ns0_clueServerStatus(response_element);

.NET Client Library

Created Aug 18, 2012 3:48:25 PM

Introduction

The .NET client-side library defines the classes that can be (de)serialized to/from XML. This is useful for accessing the REST endpoints that are published by this application.

Java Client Library

Created Aug 18, 2012 3:48:25 PM

Introduction

The Java client-side library is used to access the Web service API for this application.

The JAX-WS client-side library is used to provide the set of Java objects that can be serialized to/from XML using JAXB. This is useful for accessing the REST endpoints that are published by this application.

REST Example (Raw JAXB)

java.net.URL url = new java.net.URL(baseURL + "/game"); JAXBContext context = JAXBContext.newInstance( ClueServerStatus.class ); java.net.URLConnection connection = url.openConnection(); connection.connect(); Unmarshaller unmarshaller = context.createUnmarshaller(); ClueServerStatus result = (ClueServerStatus) unmarshaller.unmarshal( connection.getInputStream() ); //handle the result as needed...

REST Example (Jersey client)

com.sun.jersey.api.client.Client client = com.sun.jersey.api.client.Client.create(); ClueServerStatus result = client.resource(baseUrl + "/game") .get(ClueServerStatus.class); //handle the result as needed...

Objective C Client Library

Created Aug 18, 2012 3:48:24 PM

Introduction

The Objective C module generates the source code for the Objective C classes and (de)serialization functions that can be used in conjunction with libxml2 to (de)serialize the REST resources as they are represented as XML data.

The generated Objective C source code depends on the XML Reader API and the XML Writer API as well as the base OpenStep foundation classes.

REST XML Example

#import <Clue-Solver-Rest.h> //... CLUE_SOLVER_RESTNS0ClueServerStatus *responseElement; NSData *responseData; //data holding the XML from the response. NSURL *baseURL = ...; //the base url including the host and subpath. NSURL *url = [NSURL URLWithString: @"/game" relativeToURL: baseURL]; NSMutableURLRequest *request = [[NSMutableURLRequest alloc] initWithURL:url]; NSURLResponse *response = nil; NSError *error = NULL; [request setHTTPMethod: @"GET"]; //this example uses a synchronous request, //but you'll probably want to use an asynchronous call responseData = [NSURLConnection sendSynchronousRequest:request returningResponse:&response error:&error]; CLUE_SOLVER_RESTNS0ClueServerStatus *responseElement = [CLUE_SOLVER_RESTNS0ClueServerStatus readFromXML: responseData]; [responseElement retain]; //handle the response as needed...
